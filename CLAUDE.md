## How to make common edits

- **Text / copy:** edit directly in the relevant `.html` file.
- **Colors, spacing, fonts, radii:** change the **CSS custom properties (design tokens)** near the top of `css/styles.css` rather than hardcoding values — they cascade everywhere.
- **Images / video:** replace the file in `images/` or `videos/`, or change the `src`. Note `<img>` often has a `srcset` (responsive sizes) — update those too.
- **Links / nav:** page links are relative (`about.html`, `product/product-a.html`). Update them if you rename/move pages.
- **See the design system:** open `template/style-guide.html` — it renders the colors, type scale and components in one place.

## ⚠️ Critical rules (so you don't break it)

- **Do not edit `js/main.js` or `js/vendor/*`** — they are third-party/runtime libraries (minified on purpose).
- **Keep `data-w-id="…"` attributes and `w-*` classes** (`w-nav`, `w-dropdown`, `w-slider`, `w-tab`, `w-embed`, …). The runtime keys nav, sliders, tabs and **scroll/entrance animations** off them. Strip or rename them and those interactions silently break (elements can get stuck invisible).
- **`data-page-ref` on `<html>`** drives each page's load animations — leave it.
- **No templating:** the header/nav and footer are **copied into every `.html` file**. To change them site-wide, apply the same edit to every page (find-and-replace across files).
- Prefer editing **tokens over hardcoded values**; keep the existing class names when moving markup.
